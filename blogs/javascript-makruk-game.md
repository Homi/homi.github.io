**การเขียนโปรแกรมหมากรุกไทยด้วย JavaScript + HTML + CSS: เปลี่ยนลอจิกสากลสู่กระดานไทย**

การพัฒนาโปรแกรมเกมกระดานอย่าง "หมากรุกไทย" (Makruk) บนเว็บบราวเซอร์ เป็นหนึ่งในโปรเจกต์ที่ท้าทายและช่วยฝึกฝนกระบวนการคิดเชิงตรรกะ (Logical Thinking) ได้อย่างยอดเยี่ยม หัวใจสำคัญของความสำเร็จไม่ได้อยู่ที่การเขียนโค้ดให้สั้นที่สุด แต่อยู่ที่การวางโครงสร้างสถาปัตยกรรมของโปรแกรมให้เป็นระเบียบ โดยเฉพาะการแยกส่วนประกอบระหว่าง "ข้อมูลสถานะของเกม" ออกจาก "การแสดงผลบนหน้าจอ" อย่างเด็ดขาด

บทความนี้จะสรุปหลักการ เครื่องมือ และขั้นตอนในการสร้างโปรแกรมหมากรุกไทย ตั้งแต่ตารางว่างเปล่าไปจนถึงกระดานที่พร้อมประลองฝีมือ

---

### 🎯 หลักการสำคัญในการพัฒนา

1. **Separation of Concerns (การแยกส่วนการทำงาน):** ห้ามนำข้อมูลว่าหมากตัวไหนอยู่ช่องใดไปฝากไว้กับ HTML DOM โดยตรง แต่จะต้องมีตัวแปรในส่วนของลอจิก (Logic) คอยจำสถานะกระดานเสมอ การแสดงผลบนหน้าจอต้องเป็นเพียง "ภาพสะท้อน" ของข้อมูลนั้น
2. **การต่อยอดจากโครงสร้างสากล:** กระดานหมากรุกไทยมีขนาด 8x8 ช่องเท่ากับหมากรุกสากล ดังนั้นสถาปัตยกรรมพื้นฐานของระบบ (เช่น การจำลองกระดาน การสลับตาเดิน การหาช่องเป้าหมาย) สามารถประยุกต์ใช้ร่วมกันได้ สิ่งที่ต้องปรับเปลี่ยนคือ "กฎการเดินเฉพาะตัว" และ "การให้คะแนนความสำคัญของหมาก"

---

### 🛠️ เครื่องมือที่ใช้ (Tech Stack)

การสร้างหมากรุกไทยบนเว็บสามารถทำได้ด้วย 3 ทหารเสือพื้นฐาน โดยไม่จำเป็นต้องพึ่งพา Framework ที่ซับซ้อน:

*   **HTML (ส่วนโครงสร้าง):** ทำหน้าที่สร้างกรอบหรือ Container สำหรับกระดาน รวมถึงพื้นที่แสดงผลอื่นๆ เช่น แผงควบคุม สถานะเกม เมนูเลือกผู้เล่น หรือกล่องข้อความแสดงตาเดิน
*   **CSS (ส่วนตกแต่งและการวาดกระดาน):** 
    *   **การจัดการพิกัด:** ใช้ในการวาดตาราง 64 ช่อง สลับสีสว่าง-มืด โดยอาจใช้เทคนิคจัดวางตำแหน่งแบบกำหนดจุดสัมบูรณ์ (Absolute Positioning) ตามแนวแกน X และ Y 
    *   **Sprite Sheet:** เทคนิคการรวมรูปตัวหมากทุกประเภททั้งฝ่ายขาวและดำไว้ในไฟล์ภาพเดียว แล้วใช้ CSS เลือกแสดงผลเฉพาะพิกัดของหมากตัวที่ต้องการ เพื่อลดภาระการโหลดรูปภาพ
    *   **การกลับกระดาน:** รองรับการสร้างคลาสสลับมุมมองหน้าจอ สำหรับให้ผู้เล่นมองจากฝั่งหมากดำ
*   **JavaScript (ส่วนสมองกล):** เป็นหัวใจหลักในการคุมทุกอย่างของเกม 
    *   จัดการโครงสร้างข้อมูล (Data Structures) เช่น อาร์เรย์ 2 มิติ
    *   ดักจับการตอบสนอง (Event Listeners) เมื่อผู้เล่นคลิกเลือกหมาก
    *   ประมวลผลตรรกะ AI หากมีการสร้างระบบให้เล่นกับคอมพิวเตอร์

### 🚀 ขั้นตอนการพัฒนา (Step-by-Step)

**1. สร้างกระดานแสดงผล (UI Rendering)**
เริ่มต้นจากการใช้ HTML และ CSS สร้างพื้นที่ตาราง 8x8 กำหนดขนาดช่องให้เท่ากัน และสลับสีให้ถูกต้องตามมาตรฐานกระดาน พร้อมทั้งเตรียมระบบรองรับการแสดงรูปตัวหมากบนแต่ละช่อง

**2. วางโครงสร้างข้อมูล (Data Structure)**
ในส่วนของ JavaScript ให้สร้างอาร์เรย์ 2 มิติ (2D Array) ขนาด 8x8 เพื่อใช้เป็นตัวแทนของกระดานในหน่วยความจำ โดยกำหนดค่าเริ่มต้นว่าช่องใดมีหมากประเภทใดอยู่ ฝ่ายใดเป็นเจ้าของ และช่องใดเป็นช่องว่าง เมื่อข้อมูลตรงนี้อัปเดต หน้าจอจะต้องถูกสั่งให้วาดใหม่ให้ตรงกันเสมอ

**3. กำหนดกฎกติกาการเดิน (Movement Logic)**
นี่คือส่วนที่แยกระหว่างหมากรุกสากลกับหมากรุกไทย ต้องเขียนฟังก์ชันเพื่อตรวจสอบทิศทางที่อนุญาตให้เดินได้ (Valid Moves) สำหรับหมากแต่ละคลาส เช่น:
*   **ม้าและเรือ:** เดินคล้ายกติกาสากล
*   **โคน:** เดินหน้าและเฉียงรวม 5 ตา
*   **เม็ด:** เดินเฉียง 4 ทิศ
*   **เบี้ย:** เบี้ยหมากรุกไทยมีการเลื่อนยศ (เบี้ยหงาย) ในตำแหน่งที่ต่างจากสากล

**4. จัดการการโต้ตอบของผู้เล่น (User Interaction)**
เขียนระบบดักจับการคลิกเมาส์ เมื่อผู้เล่นคลิกที่ตัวหมาก โปรแกรมจะต้องคำนวณและทำไฮไลต์ช่องที่หมากตัวนั้นสามารถเดินไปได้ และเมื่อคลิกที่ช่องเป้าหมาย โปรแกรมจะตรวจสอบความถูกต้อง ย้ายข้อมูลในอาร์เรย์ และสั่งเคลียร์หมากที่ถูกกินออกจากกระดาน

**5. ระบบกติกาพิเศษและสถานะเกม (Game States)**
เพิ่มลอจิกสำหรับการตรวจสอบเงื่อนไขการจบเกม เช่น การถูก "รุก" (Check) และ "รุกจน" (Checkmate) รวมถึงข้อกำหนดที่เป็นเอกลักษณ์ของหมากรุกไทยอย่าง **"การนับศักดิ์กระดาน"** เพื่อขอเสมอในกรณีที่เข้าสู่ช่วงปลายเกมและไม่มีเบี้ยคว่ำเหลืออยู่

---

การนำโครงสร้างพื้นฐานมาดัดแปลงเป็นหมากรุกไทย ถือเป็นการผสานศิลปะของบอร์ดเกมพื้นบ้านเข้ากับศาสตร์แห่งการเขียนโปรแกรม เป็นจุดเริ่มต้นที่ดีในการทำความเข้าใจสถาปัตยกรรมซอฟต์แวร์ ซึ่งสามารถนำไปต่อยอดเป็นเกมกระดานอื่นๆ เช่น จตุรงค์ หรือสร้าง AI หมากรุกที่ฉลาดขึ้นในอนาคตได้ครับ
